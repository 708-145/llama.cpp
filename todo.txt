- [ ] **Verify nf4_xs and fp4_xs quantization modes end-to-end.**
  - **Reason:** Could not be tested due to sandbox environment limitations.
  - **Details:**
    - The process requires a base FP16 GGUF model for quantization.
    - Attempts to download a pre-existing FP16 GGUF (e.g., from TheBloke/phi-2-GGUF or Qwen/Qwen1.5-0.5B-Chat-GGUF) were unsuccessful as these repositories do not host direct FP16 GGUFs.
    - The alternative was to download an original Hugging Face model (e.g., microsoft/phi-2 in SafeTensors format) and convert it to FP16 GGUF using `convert_hf_to_gguf.py`.
    - This conversion script requires `torch` and `numpy`. While `numpy` installed successfully, `pip install torch` failed due to "No space left on device" in the sandbox.
  - **Suggested Next Steps (on a machine with sufficient disk space):**
    1. Install dependencies for model conversion: `pip install torch numpy huggingface_hub sentencepiece` (sentencepiece might also be needed by the conversion script).
    2. Download the original `microsoft/phi-2` model files (SafeTensors and JSONs) to a local directory (e.g., `models/phi-2-hf`):
       ```bash
       python download_model.py # Using the version that downloads microsoft/phi-2
       ```
    3. Convert the downloaded HF model to FP16 GGUF:
       ```bash
       python convert_hf_to_gguf.py models/phi-2-hf/ --outfile models/phi-2.F16.gguf --outtype f16
       ```
    4. Quantize `models/phi-2.F16.gguf` to `NF4_XS`:
       ```bash
       build/bin/quantize models/phi-2.F16.gguf models/phi-2.NF4_XS.gguf NF4_XS
       ```
    5. Quantize `models/phi-2.F16.gguf` to `FP4_XS`:
       ```bash
       build/bin/quantize models/phi-2.F16.gguf models/phi-2.FP4_XS.gguf FP4_XS
       ```
    6. Run inference with `llama-cli` on `models/phi-2.NF4_XS.gguf` and `models/phi-2.FP4_XS.gguf`, comparing output quality against `models/phi-2.F16.gguf` or another known good quantization (e.g., Q4_0 if also generated).
       Example inference command:
       ```bash
       build/bin/llama-cli -m models/phi-2.NF4_XS.gguf -n 32 -p "Once upon a time" -e
       ```
    7. If issues are found (crashes, garbled output, significantly degraded quality compared to FP16/Q4_0), document them for further investigation.
